# Plan Command

Create a comprehensive implementation plan for: $ARGUMENTS

## Instructions

1. **DO NOT write any code yet** - this is planning only
2. Explore the codebase to understand current structure
3. Read SPECIFICATION.md if it exists
4. Identify all files that need to be created or modified
5. List implementation steps in logical order
6. Define success criteria and verification steps
7. Identify potential risks or blockers
8. **Wait for approval before implementing**

## Output Format

```markdown
## Implementation Plan: [Feature Name]

### Overview
[Brief description of what will be built]

### Files to Create
- `path/to/new_file.py` - [purpose]

### Files to Modify
- `path/to/existing.py` - [what changes]

### Implementation Steps
1. [Step with details]
2. [Step with details]

### Dependencies
- [Any new packages needed]

### Success Criteria
- [ ] [Testable criterion]
- [ ] [Testable criterion]

### Risks
- [Potential issue and mitigation]
```

## Important
- Be thorough but concise
- Consider edge cases
- Think about testing strategy
- DO NOT start implementing until plan is approved
